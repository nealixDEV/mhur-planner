Implement a global Team Role Bonus system.

The Team Role Bonus system is NOT a standalone calculator. It is a global modifier that automatically affects every calculator in the planner.

Global System
━━━━━━━━━━━━━━━━━━━━

This system should not perform calculations itself.

Instead, it should maintain a single global Team Role Bonus state that every calculator automatically reads.

Whenever any of the following changes:

• Selected Character
• Enable Team Role Bonuses
• Teammate 1
• Teammate 2

Recalculate the active Team Role Bonuses once and update the global state.

Every calculator should automatically include those bonuses without requiring any additional user input.

━━━━━━━━━━━━━━━━━━━━
Integration
━━━━━━━━━━━━━━━━━━━━

When Team Role Bonuses are enabled, they should automatically modify every applicable calculator, including:

• Damage Calculator
• HP Calculator
• HP / Defense Calculator
• Reload Calculator
• Movement Calculator
• Misc Calculator
• Ability Manifest Calculator (if applicable to future role effects)

The user should never need to enable role bonuses separately inside each calculator.

━━━━━━━━━━━━━━━━━━━━
Architecture
━━━━━━━━━━━━━━━━━━━━

Create a centralized TeamRoleBonusManager (or equivalent shared state).

Responsibilities:

• Determine the selected character's role.
• Count teammate roles.
• Determine each active Team Role Bonus level.
• Store every active Team Role Bonus globally.
• Provide those bonuses to every calculator.

All calculators should consume this shared Team Role Bonus state instead of implementing their own team bonus logic.

This ensures:

• One source of truth.
• No duplicated calculations.
• Every calculator remains synchronized.
• Future Team Role Bonus value changes only require updating one data table.
